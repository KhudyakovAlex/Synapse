# User Experience Language

Язык описания интерфейсов для UX-дизайна в MD-файлах.

Служит для последующего рендеринга в HTML.

## Структура UXL-блока в MD-файле

Начало блока UXL: ```UXL  

Окончание блока UXL: ```  

Содержимое блока начинается с задания размера окна. Размер задается в пикселях в формате WxH (ШИРИНАxВЫСОТА). Если размер не указан, считаем 500x500.  

Каждый новый тег элемента начинается с новой строки.  

Вложенность элементов определяется отступами. Каждый уровень вложенности добавляет два пробела к отступу предыдущего уровня.  

В корне блока — только теги P.

## Теги элементов

P - Страница  
F - Поле  
B - Кнопка  
C - Надпись    
T - Таблица  
TC - Форматы колонок  
TH - Ячейки заголовка  
TD - Ячейки с данными  

## Формат тегов

### Формат тегов по умолчанию

TAG\ID\CAPTION\SIZE\ALIGN\ACTION\HINT  

где составляющие:  
- TAG - тег элемента
- ID - ID для идентификации внутри блока UXL
- CAPTION - строка, отображаемая у некоторых элементов в рендере
- SIZE - размеры элемента
- ALIGN - выравнивание элемента
- ACTION - действие при клике на элемент
- HINT - подсказка, всплывающая при наведении курсора на элемент

#### ID
Строка из латинских символов, цифр, подчеркивания, дефиса.
Должен быть уникальным в пределах типа тега. Например, могут существовать одновременно P1 и F1, но не может быть двух P1.

#### CAPTION
Строка: надпись на кнопке, содержимое надписи и т.п.

#### SIZE
Ширина и высота элемента в формате: WxH (ШИРИНАxВЫСОТА).  
W и H - пиксели (если с px (330px) или без единиц (330)) или проценты от внутренних габаритов элемента-родителя (если с %: 33%).  
Если размер не указан, он определяется содержимым элемента.

#### ALIGN
Выравнивание по горизонтали и вертикали в пределах элемента-родителя в формате: LRTB.
L - привязка к левому краю,  
R - привязка к правому краю, 
LR - растягивает элемент во всю допускаемую элементом-родителем ширину.  
Если не указаны ни L ни R, выравнивание по горизонтали по середине.
T - привязка к верхнему краю,  
B - привязка к нижнему краю, 
TB - растягивает элемент во всю допускаемую элементом-родителем высоту.  
Если не указаны ни T ни B, выравнивание по вертикали по середине.

#### ACTION
Одна из команд:
- GOTO:TID - переход на страницу, где TID = тег + ID (например, GOTO:P2 означает переход на страницу с тегом P и ID=2).

### Порядок составляющих

Составляющие кроме тега могут отсутствовать.
Составляющие должны быть на своих местах, т.е., если какая-то составляющая отсутствует, её место должно быть занято пустым значением (два обратных слэша подряд \\ означают пустое значение).
Если после составляющего других его коллег нет, то и \ не нужны.
После тега обязателен один \.
Пример: B\12\Кнопка\\LRTB\GOTO:P2 (здесь SIZE отсутствует, поэтому два слэша подряд)

### Формат тегов таблицы TC

TC\WA\WA\WA\...

где:
- W - ширина колонки в %
- A - выравнивание в ячейке колонки по горизонтали: L или R; если не указано - выравнивание по середине
Сумма процентов всех колонок должна равняться 100%. Если сумма отличается от 100%, колонки пропорционально масштабируются до 100%.
Пример: TC\20R\70L\10

### Формат тегов таблицы TH и TD

TH(или TD)\C\C\C\...

где:
- C - текстовое содержимое ячейки
Пример: TD\100\Жигулевское\- (здесь "-" - это текст в ячейке, а не пустое значение)

### Формат тегов P

P\ID\CAPTION

где:
- ID - Строка из латинских символов, цифр, подчеркивания, дефиса; должен быть уникальным в пределах типа тега.
- CAPTION - Заголовок страницы

## Рендеринг

В итоговом рендере должна по умолчанию показываться первая страница в блоке.
