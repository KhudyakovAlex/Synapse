# Приложение. База данных

АПК Синапс v1.0. ПО. Спецификации на разработку

**Последнее изменение:** 05.12.2025

## 1. Термины и определения

1.1. **Рабочие** данные — необходимые для организации работы освещения и в прошивке, и в приложении. В приложении находятся в таблицах SQLite, в прошивке в виде массивов структур C.

1.2. **Интерфейсные** данные — используемые только приложением для организации пользовательского интерфейса.  Примеры: названия светильников, номера их иконок.. В приложении находятся в таблицах SQLite, в прошивке в виде единого блока где-то в постоянной памяти.

## 2. Основные положения

2.1. База данных в приложении представлена в виде схемы в SQLite.

2.2. Задача: иметь в ней копию базы из прошивки контроллера, расширенную интерфейсными данными. В прошивке интерфейсные данные лежат в отдельной области. Здесь они должны быть разложены по дополнительным полям в таблицах.

2.3. В отличие от контроллера здесь полноценная таблица с контроллерами, т.к. телефон может с несколькими работать.

2.4. В целях подстроиться максимально под структуру БД контроллера таблицы здесь (кроме CONTROLLERS) имеют на постоянке столько записей, сколько элементов имеют соответствующие массивы в прошивке: в прошивке в массиве 64 элемента под светильники и тут 64 записи. Записи эти в таблицах не создаются и не удаляются, а просто заполняются данными из соответствующих массивов прошивки.

2.5. Значения в полях ID этой базы (кроме CONTROLLERS) соответствуют индексам соответствующих элементов в соответствующих массивах БД прошивки.



## 3. Таблицы

**Рабочие** данные помечены в таблицах ниже жирным.

### 3.1. CONTROLLERS

Таблица с информацией о контроллерах, к которым коннектились и которые не удалили. Чтобы видеть контроллеры, с которыми в данный момент не на связи, и чтобы не вводить пароли, если уже их правильно вводили.

- ID (INTEGER, PK)
- SCALE (INTEGER) — масштаб сетки с иконками локаций на странице контроллера;
- **NAME** (TEXT) — название контроллера, показываемое в приложении и списке устройств Bluetooth;
- **PASSWORD** (TEXT) — пароль заводской или изменённый пользователем; 4 цифры; заводской на коробке (или наклейке на корпусе);
- **IS_SCHEDULE** (TEXT) — включена ли в контроллере работа по расписанию (T/F);
- **IS_AUTO** (TEXT) — главный выключатель автоматического режима для всего контроллера (T/F);
- **ICO_NUM** (INTEGER) — номер иконки, которую будет иметь контроллер в приложении; 0 — дефолтная.
- **STATUS** (TEXT) — текущее состояние контроллера: A (ACTIVE) — штатная работа, I (INIT) — инициализация линии DALI, F (FAILURE) — неполадка.
- **SCENE_NUM** (INTEGER) - световая сцена, включенная в для всего объекта (0,1,2,3,4)

### 3.2. LOCATIONS

Локации.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- NAME (TEXT) — название локации;
- ICO_NUM (INTEGER) — номер иконки, которую будет иметь локация в приложении; 0 — дефолтная;
- SCALE (INTEGER) — масштаб сетки с иконками светильников на странице локации;
- POS_X (INTEGER) — позиция слева направо по X в сетке локаций контроллера;
- POS_Y (INTEGER) — позиция сверху вниз по Y в сетке локаций контроллера;
- **EXIST** (TEXT) - задействована ли запись в работе или это пустое место где может храниться запись (T/F)
- **IS_AUTO** (TEXT) — работает ли локация в автоматическом режиме по датчикам (T/F).
- **SCENE_NUM** (INTEGER) - световая сцена, включенная в этой локации (0,1,2,3,4)

### 3.3. GROUPS

Группы светильников.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- NAME (TEXT) — название группы;
- **EXIST** (TEXT) - задействована ли запись в работе или это пустое место где может храниться запись (T/F)
- **LOCATION_ID** (INTEGER) — локация, к которой привязана группа; -1 — группа без локации (в корне);
- **DALI_NUM** (INTEGER) — номер группы в DALI.
- **SCENE_NUM** (INTEGER) - световая сцена, включенная в этой группе (0,1,2,3,4)


### 3.4. LUMINAIRES

Светильники.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- NAME (TEXT) — название светильника;
- ICO_NUM (INTEGER) — номер иконки, которую будет иметь светильник в приложении; 0 — дефолтная;
- POS_X (INTEGER) — позиция слева направо по X в сетке светильников локации;
- POS_Y (INTEGER) — позиция сверху вниз по Y в сетке светильников локации;
- **EXIST** (TEXT) - задействована ли запись в работе или это пустое место где может храниться запись (T/F)
- **DALI_ADDR** (INTEGER) — короткий адрес DALI;
- **LOCATION_ID** (INTEGER) — локация, к которой привязан светильник; -1 — светильник без локации (в корне);
- **GROUP_ID** (INTEGER) — группа, к которой привязан светильник; если светильник не в группе: -1;
- **VAL_BRIGHT** (INTEGER) — значение яркости;
- **VAL_TW** (INTEGER) — значение температуры белого света;
- **VAL_R** (INTEGER) — значение красного канала (для RGB);
- **VAL_G** (INTEGER) — значение зелёного канала (для RGB);
- **VAL_B** (INTEGER) — значение синего канала (для RGB);
- **VAL_W** (INTEGER) — значение белого канала (для RGBW).
- **SCENE_NUM** (INTEGER) - световая сцена, включенная у этого светильника (0,1,2,3,4)

### 3.5. SCENE_LUMINAIRES

Параметры светильников в сценах.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- **SCENE_NUM** (INTEGER) - световая сцена (0,1,2,3,4)
- **LUMINAIRE_ID** (INTEGER) — светильник;
- **VAL_BRIGHT** (INTEGER) — значение яркости;
- **VAL_TW** (INTEGER) — значение температуры белого света;
- **VAL_R** (INTEGER) — значение красного канала (для RGB);
- **VAL_G** (INTEGER) — значение зелёного канала (для RGB);
- **VAL_B** (INTEGER) — значение синего канала (для RGB);
- **VAL_W** (INTEGER) — значение белого канала (для RGBW).


### 3.6. PRES_SENSORS

Датчики присутствия.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- NAME (TEXT) — название датчика;
- **EXIST** (TEXT) - задействована ли запись в работе или это пустое место где может храниться запись (T/F)
- **DALI_ADDR** (INTEGER) — короткий адрес DALI датчика;
- **DALI_INST** (INTEGER) — номер инстанса датчика движения в DALI;
- **LOCATION_ID** (INTEGER) — локация, к которой привязан датчик; -1 — датчик без локации (в корне);
- **ACTION_OCCUPANCY_ID** (INTEGER) — действие, повешенное на присутствие; -1 — не повешено действие;
- **ACTION_VACANCY_ID** (INTEGER) — действие, повешенное на отсутствие; -1 — не повешено действие;
- **DELAY** (INTEGER) — задержка HOLD_TIME.

### 3.7. BRIGHT_SENSORS

Датчики освещённости.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- NAME (TEXT) — название датчика;
- **EXIST** (TEXT) - задействована ли запись в работе или это пустое место где может храниться запись (T/F)
- **DALI_ADDR** (INTEGER) — короткий адрес DALI датчика;
- **DALI_INST** (INTEGER) — номер инстанса датчика движения в DALI;
- **LOCATION_ID** (INTEGER) — локация, к которой привязан датчик; -1 — датчик без локации (в корне);
- **GROUP_ID** (INTEGER) — группа, к которой привязан датчик; -1 — группа не назначена.

### 3.8. BUTTON_PANELS

Кнопочные панели.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- NAME (TEXT) — название панели;
- **EXIST** (TEXT) - задействована ли запись в работе или это пустое место где может храниться запись (T/F)
- **DALI_ADDR** (INTEGER) — короткий адрес панели в DALI;
- **LOCATION_ID** (INTEGER) — локация, к которой привязана панель; -1 — панель без локации (в корне);

### 3.9. BUTTONS

Кнопки.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- NAME (TEXT) — название кнопки на панели;
- **EXIST** (TEXT) - задействована ли запись в работе или это пустое место где может храниться запись (T/F)
- **BUTTON_PANEL_ID** (INTEGER) — кнопочная панель, в которой кнопка расположена;
- **DALI_INST** (INTEGER) — номер инстанса кнопки в DALI;
- **ACTION_SET_SHORT_ID** (INTEGER) — действие по короткому нажатию; -1 — не повешено действие;
- **ACTION_LONG_ID** (INTEGER) — действие по долгому нажатию; -1 — не повешено действие.

### 3.10. ACTIONS

Действия с устройствами.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- **EXIST** (TEXT) - задействована ли запись в работе или это пустое место где может храниться запись (T/F)
- **ACTION_SET_ID** (INTEGER) — привязка к набору действий; если действие не входит ни в какой набор: -1;
- **POS** (INTEGER) — номер по порядку действия (начиная с 1) в списке действий, привязанных к кнопке (случай перебора действий одной кнопкой)


### 3.11. SUBACTIONS

Поддействия с устройствами (команды для пользователя).

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- **EXIST** (TEXT) - задействована ли запись в работе или это пустое место где может храниться запись (T/F)
- **ACTION_ID** (INTEGER) — действие;
- **OBJECT_TYPE** (INTEGER) — тип объекта изменений:
    - 1 — весь объект;
    - 2 — локация;
    - 3 — группа;
    - 4 — светильник;
- **OBJECT_NUM** (INTEGER) — номер объекта изменений:
    - 1 — сцена;
    - 2 — температура TW;
    - 3 — флаг АВТО.
- **VALUE** (INTEGER) — значение в зависимости от объекта:
    - сцена - 0..4;
    - температура TW - 0..254;
    - флаг АВТО - 0 / 1.

### 3.12. ACTION_SETS

Наборы действий для навешивания на короткие нажатия кнопок (когда перебираем действия одной кнопкой).

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- **EXIST** (TEXT) — задействована ли запись в работе или это пустое место, где может храниться запись (T/F)

### 3.13. EVENTS

События расписания.

- ID (INTEGER, PK) — идентификатор в соответствующем массиве в прошивке;
- **EXIST** (TEXT) — задействована ли запись в работе или это пустое место, где может храниться запись (T/F)
- **DAYS** (TEXT) — 'FFTFFFF' — событие по дням недели; если все F — ежедневное;
- **TIME** (TEXT) — время события с точностью до минуты в формате HHMM 24H;
- **SMOOTH** (TEXT) — плавное изменение параметров от предыдущих значений (яркость, температура света) (T/F);
- **ACTION_SET_ID** (INTEGER) — набор действий; -1 — не назначен.
