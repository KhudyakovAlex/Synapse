2. Идеальный вариант работы

2.1. Пользователь при заходе в приложение видит в списке контроллеров все контроллеры Синапс на связи.

2.2. Контроллеры, в которые приложение заходило раньше, но которых в данный момент нет на связи, не показываем.

2.3. Если при последнем выходе из приложения оно находилось в состоянии работы с конкретным контроллером и этот контроллер сейчас на связи, сразу заходим в этот контроллер.

2.4. Контроллеры уже сконфигурированные, у которых были заменены заводские имена, отображаются новыми именами: "Этаж 2", "Моя лачуга" и т.п.

2.5. По умолчанию имена контроллеров, видимые в списке контроллеров, должны иметь вид SYNAPSE XXXXXXXX, где XXXXXXXX - серийный номер устройства.

2.6. Кроме имени у контроллера может быть изменена иконка.

2.7. В имени контроллера как-то кодируется, и пользователю в имени не показывается, номер иконки. Например: "Наш офис[5]" — пользователь должен видеть только "Наш офис" и иконку под номером 5, которая берётся из библиотеки иконок приложения.

2.8. Если номера иконки приложение не получило, выводится дефолтная с номером 0.

2.9. В идеале номер иконки лучше передавать другим образом.

2.10. Если в момент нахождения приложения в контроллере пропадает связь с контроллером, пользователю выдаётся модальное сообщение об этом с кнопкой выхода в список контроллеров.

2.11. В режиме пропавшей связи приложение находится неограниченно долго, пытаясь восстановить связь.

2.12. Если приложение закрывается в режиме пропавшей связи, то при следующем его открытии пытается зайти в контроллер, а если связи таки нет, показывает список контроллеров.

2.13. Контроллер как блютус-устроцство должен подключаться без пароля, но при заходе в контроллер, с которым есть связь, у пользователя для дальнейших действий с устройством должен запрашиваться пароль (пинкод 4 цифры).
